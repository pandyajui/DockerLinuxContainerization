Containerization

Containerization is a fake OS that's only used when you want to run your code. Container has one kernel which is host OS and one kernel cannot be divided into several kernel machines. You can run a working server in a container. Containerization does not need a "Guest OS" it can run via "Host OS". Guest OS is part of a Virtual Machine it has all attributes that an Host OS has. Container contains all the run time, libraries, and binaries to run an application. Containers ability to package up a lot of dependencies from the OS in one container does not have much OS dependencies. As for containers we stick to stick with physical infrastructure. Benefits of Containerization:

  • Increased Portability: Containers can be moved across one or many a servers and this does not slow down activities running in the traditional computing environment.
  • Improved Scalability: You can add or remove features
  • Simple and Fast Deployment: Speeding up deployment and config processes are an advantage to containerization.
  • Enhanced Productivity: Helps simplify the installation processes and lessen dependency error which saves a lot of time when installing applications.
  • Improved Security: Isolation allows to improvise security by setting up each applications larger processes apart from one & another in different containers.
